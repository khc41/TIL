# 📅 2026-03-19
## 📚 읽은 개발 서적:
### 8장. 분산 시스템의 골칫거리
- 어떤 것이든지 잘못될 가능성이 있다면 잘못된다고 가정한다.
- 우리는 엔지니어로서 모든 게 잘못되더라도 제 역할을 해내는 시스템을 구축하는 것이다.

#### 결함과 부분 장애
- 단일 컴퓨터일 경우 프로그램은 돌아가거나 안 돌아가거나 둘 중 하나이다.
- 좋은 소프트웨어가 설치된 컴퓨터는 완전하게 동작하거나 전체 장애가 발생하는데 이는 잘못된 결과를 반환하기 보다는 완전히 동작하지 않게끔 의도적으로 설계된 것이다.
- 네트워크로 연결된 여러 컴퓨터에서 실행되는 분산 시스템에서는 이상화된 시스템 모델이 동작하지 않고 물리적 세계의 지저분한 현실을 마주하게 된다.
- 어떤 부분은 잘 동작하지만 다른 붑누은 예측할 수 없는 방식으로 고장나는 것을 부분 장애라 하는데 비결정적이라 어렵다.

##### 클라우드 컴퓨팅과 슈퍼컴퓨팅
- 고성능 컴퓨팅과 클라우드 컴퓨팅이 양 극단에 있고 전통적인 기업형 데이터센터는 중간 지점에 있다.
- 슈퍼컴퓨터
  - 계산 상태를 지속성 있는 저장소에 체크포인트로 저장한다.
  - 노드 하나에 장애가 발생했을 때 전체 클러스터 작업부하를 중단하고, 복구된 후 마지막 체크포인트부터 계산으 ㄹ재시작한다.
- 인터넷 서비스를 구현하는 시스템
  - 언제라도 사용자에게 지연 시간이 낮은 서비스를 제공해야 한다는 점에서 온라인이다.
  - 슈퍼컴퓨터는 노드 사이에 공유 메모리와 원격 직접 메모리 접근을 사용해 통신하지만, 클라우드 서비스는 상용 장비를 사용해 구축하므로 동일한 성능을 제공하지만 실패율도 높다.
  - 거대한 데이터센터의 네트워크는 IP와 이더넷을 기반으로 하며 높은 양단 대역폭을 제공하기 위해 클로스 토폴로지로 연결돼있고, 슈퍼컴퓨터는 통신 패턴이 정해진 작업부하에서 높은 성능을 보여주는 다차원 메시나 토러스 같은 특화된 네트워크 토폴로지를 자주 사용한다.
  - 시스템이 커질수록 구성 요소 중 하나가 고장날 가능성이 높아지므로 수천 개의 노드가 있는 시스템에서는 항상 뭔가 고장난 상태라고 가정하는 게 합리적이다.
  - 시스템이 장애가 난 노드를 감내할 수 있고 전체적으로는 계속 동작할 수 있다면 이는 운영과 유지보수에 매우 유용한 특성이 된다.
  - 지리적으로 분산된 배포를 할 때 인터넷을 거치면 로컬 네트워크에 비해 느리고 신뢰성도 떨어진다.
- 분산 시스템이 동작하게 만들려면 부분 장애 가능성을 받아들이고 소프트웨어에 내결함성 메커니즘을 넣어야 한다.
- 작은 시스템도 결함이 생길 것이고 결함을 처리해야하기 때문에 결함 처리는 소프트웨어 설계에 일부여야 한다.
- 신뢰성 없는 구성 요소를 사용해 신뢰성 있는 시스템 구축하기
  - 오류 수정 코드는 무선 네트워크에서 일부 비트가 잘못되는 통신 채널을 통해 정확히 전송할 수 있게 해준다.
  - IP는 신뢰성이 없고 TCP는 IP 위에서 더욱 신뢰성이 높은 전송 계층을 제공한다.
